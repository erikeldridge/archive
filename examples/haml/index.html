Credit

  • Inspired by Derek Gathright’s talk on YQL. He created this presentation-style, developed the HTML/CSS/JS, and defined the UI

Syntax

* Slides are delimited by "- - -" * The title slide is assumed to be 1st * If there's an h3 tag in the slide, it's stuck in the header

Organization

  • template.html defines the presentation wrapper and provides template html for each slide
  • slides.textile defines slide contents
  • build.rb renders textile, parses the template html, and maps the textile to template
  • the css directory contains css for preso display and syntax highlighting css
  • the css directory contains js for preso control and syntax highlighting js

Textile

  • Textile balances minimalist syntax & expressibility (CSS/tag access)
  • There’s a full-featured Ruby parser for Textile (RedCloth)
  • Allows embedded html/css/js

Ruby

  • Ruby is widely available
  • Ruby is fun to work with

HTML5

  • Not quite sure why HTML5 is necessary
  • <header> and <section> tags

CSS3

  • Gives us the awesome table layout control
